# 文档导航

> 目标：把“先看什么、各文档负责什么、哪些是当前口径”一次说清，降低新会话恢复成本。

---

## 先看哪几份

### 1. 快速恢复上下文

1. `项目总览与路线图.md`
2. `progress/current_snapshot.md`
3. `../AGENTS.md`

适用场景：

- 想先知道这个项目要做什么
- 想知道当前最高优先级是什么
- 想在 1 分钟内恢复协作上下文

### 2. 第一次系统了解项目

1. `项目总览与路线图.md`
2. `spec/技术方案_v7.9.md`
3. `architecture/数据流设计.md`
4. `progress/完成度清单.md`

适用场景：

- 第一次接手项目
- 想搞清产品目标、主闭环、数据结构和实现范围

### 3. 继续当前开发

1. `progress/current_snapshot.md`
2. `progress/系统审视与问题跟进.md`
3. `progress/问题复盘与防再犯清单.md`
4. 对应专题文档

适用场景：

- 继续上一轮工作
- 修 bug、补验证、推进主流程

## 文档地图

### 根文档

- `项目总览与路线图.md`
  - 作用：项目总览层，统一回答目标、主闭环、阶段策略和近期优先级。
  - 状态：当前有效。

- `受限环境改动同步规范.md`
  - 作用：受限机器或公司电脑下的改动同步与交接规则。
  - 状态：当前有效。

### `spec/`

- `spec/技术方案_v7.9.md`
  - 作用：完整产品与技术设计蓝图。
  - 状态：基础总方案，仍有效；具体执行优先级以 `项目总览与路线图.md` 和 `progress/current_snapshot.md` 为准。

### `progress/`

- `progress/current_snapshot.md`
  - 作用：当前会话恢复入口，记录最新工作状态、环境、风险和下一步动作。
  - 状态：最高优先级的当前进度口径。

- `progress/当前主线与非主线边界.md`
  - 作用：明确当前轮真正要继续做什么，什么只保留文档和边界。
  - 状态：当前有效。

- `progress/系统待办优先级_产品版.md`
  - 作用：从产品成熟度视角列出还缺什么、为什么先做它。
  - 状态：当前有效。

- `progress/完成度清单.md`
  - 作用：按成熟度矩阵看当前哪些稳定、哪些粗糙、哪些未完成。
  - 状态：当前有效。

- `progress/系统审视与问题跟进.md`
  - 作用：系统级问题审计、验证状态和跟进清单。
  - 状态：当前有效。

- `progress/问题复盘与防再犯清单.md`
  - 作用：真实踩坑记录、预警信号和防再犯规则。
  - 状态：当前有效。

- `progress/成熟度差距与数据质量分析.md`
  - 作用：从成熟系统视角评估现阶段完成度与数据质量短板。
  - 状态：阶段性分析文档，适合做中期判断，不替代最新快照。

- `progress/相对主线变更说明_2026-03-19.md`
  - 作用：相对基线的一次性变更总说明，便于快速了解最近一大轮改动主题。
  - 状态：历史说明文档，保留参考，不作为长期唯一真相源。

### `architecture/`

- `architecture/数据流设计.md`
  - 作用：核心数据流与三层结构设计。
  - 状态：当前有效。

- `architecture/AI分析飞轮.md`
  - 作用：AI 分析从理论到反哺训练的闭环设计。
  - 状态：当前有效。

- `architecture/知识树笔记与错题挂载路线图.md`
  - 作用：知识树主线、错题挂载与笔记系统收口方案。
  - 状态：当前有效。

- `architecture/知识树产品定义.md`
  - 作用：明确“一个知识点 = 一篇 Markdown 笔记”的产品心智与展示规则。
  - 状态：当前有效。

- `architecture/AI移交设计说明.md`
  - 作用：明确当前 AI 相关工作哪些只保留为文档和接口边界，供外部团队接手。
  - 状态：当前有效。

- `architecture/Codex高频分析执行方案.md`
  - 作用：Codex 高频分析 worker 的正式执行方案和边界。
  - 状态：设计参考文档，当前轮不再作为主开发项。

- `architecture/Codex分析执行手册.md`
  - 作用：面向实际操作的 bundle 导出、交给 Codex、结果回写手册。
  - 状态：设计参考文档，当前轮不再作为主开发项。

- `architecture/AI进化路线图.md`
  - 作用：AI 能力的长期路线图。
  - 状态：长期蓝图，当前轮按“移交设计”理解，不作为主开发优先级。

### `decisions/`

- `decisions/设计决策记录.md`
  - 作用：重要设计取舍和决策背景沉淀。
  - 状态：当前有效。

### 仓库外但强相关

- `../AGENTS.md`
  - 作用：默认协作规则、产品守则、验证纪律和会话恢复标准。
  - 状态：当前有效，继续开发前建议先读。

- `../analysis-worker/ARCHITECTURE.md`
  - 作用：分析服务早期架构设计稿。
  - 状态：保留历史背景；当前执行口径优先看 `architecture/Codex高频分析执行方案.md` 和 `architecture/Codex分析执行手册.md`。
