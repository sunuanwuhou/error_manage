# 总体执行总规划（2026-04-01）

## 一、项目当前唯一主线

当前项目不再分叉，唯一主线定为：

真题导入 → 题库入库 → 刷题作答 → 自动判分 → `error_analysis` 式错题工作台 → 笔记/知识树联动 → 复盘训练

---

## 二、今天已经明确的核心判断

### 1. 主线方向已经对了
现在的问题不是“方向错了”，而是每一段还要从：
- 能用
继续收成：
- 默认顺着走
- 数据可信
- 可持续复盘

### 2. 最大价值不再是单个页面，而是主链闭环
当前真正值钱的，不是多加页面，而是让这几段真正闭环：
- 导入成功后直接可练
- 坏题进入待修复池并可修后放行
- 做完题后不只是看结果，而是进入复盘流
- 错题工作台状态开始反向影响训练调度
- 后续补上“作答过程思维采集层”

### 3. 只看错题答案，不足以做高质量错因分析
今天已经明确：
- 只知道“用户选错了什么”
- 并不能知道“用户为什么错”

所以后续必须补：
- 作答过程采集层
- 先从画布 + 规则解析入手
- 再逐步接更强的分析能力

## 四、当前阶段真正还缺什么

### 1. `error_analysis` 正式模块还未完整迁入
现在是“方向上靠拢 + 局部兼容实现”，但还不是正式整块迁入。

### 2. 兼容存储层仍不是正式表结构
当前笔记 / 知识点挂接 / 状态流仍是兼容存储，后续必须收口到正式结构。

### 3. 复盘流还是第一版
现在已经能顺着走，但还没有完整接：
- 本轮上下文
- 题目做题过程
- 更强的复盘建议

### 4. 作答过程层完全没接上
今天已经明确这是后续最值钱的增强之一。

## 五、后续总执行顺序

### 第一阶段：把当前主链收稳
目标：
- 导入 → 练习 → 结果 → 错题工作台 → 复盘流
这一整条链更加顺滑、默认可走

继续收的重点：
1. `error_analysis` 错题模块继续并线
2. 兼容存储层迁正式结构
3. 复盘流接更多真实上下文

### 第二阶段：接“作答过程思维采集层”
目标：
- 不再只看最终答案
- 开始采集解题过程

具体：
1. 每题支持过程画布
2. 保存画布轨迹 + 快照
3. 题后采集极简过程标签
4. 规则解析第一版

### 第三阶段：让过程数据反向进入诊断与训练
目标：
- 错因分析不再只是结果推测
- 开始基于过程证据做诊断

具体：
1. 画布过程规则化
2. 错因标签升级
3. 训练调度进一步按过程证据排序
4. 复盘建议更细化
