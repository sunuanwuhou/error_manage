# 规则接入说明

## 目标
把“唯一主模型、唯一主链、提交前门禁”从口头约定变成仓库内的固定规则。

## 本包包含
- `docs/rules/开发铁规.md`
- `docs/contracts/主链映射表.md`
- `docs/checklists/提交前检查清单.md`
- `.githooks/pre-push`
- `scripts/setup-githooks.sh`
- `.github/workflows/quality-gates.yml`
- `.github/PULL_REQUEST_TEMPLATE.md`

## 接入步骤
1. 将本包内容覆盖到仓库根目录。
2. 执行：
   ```bash
   chmod +x .githooks/pre-push scripts/setup-githooks.sh
   ./scripts/setup-githooks.sh
   ```
3. 本地先跑：
   ```bash
   npm run typecheck
   npm run smoke:backend
   ```
4. 提交 PR 时按模板补齐主链映射与回归结果。

## 建议执行顺序
- 先接规则与门禁
- 再继续功能开发
- 以后任何“最终版”都以 typecheck / smoke 通过为准
