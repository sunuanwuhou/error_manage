# 兼容记录层实现规格（2026-04-01）

## 一、定位
兼容记录层是系统底座。
它的任务不是“多存几个字段”，而是把题目、作答、判分、过程、错因、复训统一到同一条记录关系中。

如果没有这一层，后面每加一个题型、每接一个新能力，字段都会继续发散。

---

## 二、目标
兼容记录层要解决以下统一问题：
- 真题导入与手动录题统一
- 不同题型统一
- 不同答案结构统一
- 不同判分方式统一
- 过程记录与结果记录统一挂接
- 规则分析与 AI 分析统一挂接

---

## 三、核心实体

### 1. QuestionRecord
题目主记录。

### 2. AttemptRecord
每次作答记录。

### 3. ScoreRecord
一次作答的判分结果。

### 4. ProcessRecord
用户作答时留下的过程证据。

### 5. ErrorAnalysisRecord
错因分析结果。

### 6. ReviewTaskRecord
复训任务记录。

---

## 四、统一主键关系
建议：
- `questionId`：题目唯一 id
- `attemptId`：一次作答唯一 id
- `scoreId`：一次判分唯一 id
- `processId`：一次过程记录唯一 id
- `analysisId`：一次错因分析唯一 id
- `reviewTaskId`：一次复训任务唯一 id

关系建议：
- 一个 `questionId` 可对应多个 `attemptId`
- 一个 `attemptId` 对应一个主 `scoreId`
- 一个 `attemptId` 可关联多个 `processId`
- 一个 `attemptId` 可关联多个 `analysisId`（支持版本）
- 一个 `analysisId` 可生成多个 `reviewTaskId`

---

## 五、题型兼容策略

### 统一字段层
所有题型都保留：
- `module`：言语/判断/数量/资料/公基/申论
- `questionType`：单选/多选/填空/计算/排序/配对/主观
- `answerStructure`：choice / text / number / multi-part / subjective

### 答案字段建议
不要所有题都只用一个 `answer` 字段。
建议拆成：
- `standardAnswer`
- `userAnswer`
- `normalizedAnswer`
- `answerMeta`

这样可以兼容：
- 单选：`A`
- 多选：`[A,C,D]`
- 填空：`"24"`
- 计算：`{"value": 24, "unit": "%"}`
- 主观：长文本

---

## 六、来源兼容策略

### sourceType
- past_exam
- manual_entry
- imported_doc
- imported_image
- generated_variant

### sourceMeta
至少包含：
- 来源名称
- 年份
- 省份/考试类型
- 原始文件 id
- 页码/题号
- OCR 状态（如果有）

---

## 七、记录分层原则

### 题目层：长期稳定
保存题目的客观属性。

### 作答层：一次一条
同一题目可多次作答，不覆盖。

### 判分层：允许重复生成
规则升级、标准答案修订后，允许重新判分。

### 错因层：允许版本化
后续 AI、规则、人工修订都可能得到不同版本。

### 过程层：只追加，不覆盖原始轨迹
原始证据要尽量保留。

---

## 八、前端读取原则
前端不要直接拼接散字段。
应统一走一个聚合视图，例如：
- `QuestionFullView`
- `AttemptFullView`
- `ErrorReviewView`

由聚合层把：
- 题目
- 用户答案
- 判分
- 过程
- 错因
- 复训

一次性组织给前端。

---

## 九、存储建议
如果当前阶段使用 IndexedDB，可按 store 拆分：
- `questions`
- `attempts`
- `scores`
- `process_records`
- `error_analyses`
- `review_tasks`
- `knowledge_links`

并建立基础索引：
- `questionId`
- `attemptId`
- `module`
- `questionType`
- `isWrong`
- `createdAt`
- `updatedAt`
- `knowledgeNodeIds`

---

## 十、兼容升级要求
新增能力时，必须遵循两条：
- 优先扩展 `meta` 和 version 字段，不轻易打断主字段
- 聚合读取层兜底旧版本结构

否则后面版本一多，旧数据就会失效。
