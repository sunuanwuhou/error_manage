# 主链映射表

## 总链路

`ImportJob -> Question -> PaperPracticeSession / PracticeRecord -> UserError`

---

## 1. 导入链

### 目标
把原始文件转成正式题库题目。

### 主对象
- 入口：`ImportJob`
- 结果：`Question`

### 读取
- 文件元数据
- OCR / DOC 解析结果
- 导入确认参数

### 写入
- `ImportJob`
- `Question`

### 规则
- 导入阶段只能写导入相关状态和候选题目数据
- 正式入库后，题目统一进入 `Question`
- 禁止导入链自行新建第二套题目主表

---

## 2. 题库链

### 目标
管理正式题目资产。

### 主对象
- `Question`

### 读取
- `Question`
- 题目来源字段
- 题型、地区、年份等过滤条件

### 写入
- `Question`

### 规则
- 题库相关页面只能围绕 `Question` 展开
- 不允许题库后台直接依赖错题表做主展示

---

## 3. 练习链

### 目标
记录用户做题过程与结果。

### 主对象
- 套卷模式：`PaperPracticeSession`
- 单题 / 常规练习：`PracticeRecord`

### 读取
- `Question`
- 当前会话状态
- 用户历史记录

### 写入
- `PaperPracticeSession`
- `PracticeRecord`

### 规则
- 判题依据统一来自 `Question.answer`
- 会话进度统一写入 `PaperPracticeSession`
- 做题结果统一收口到 `PracticeRecord`

---

## 4. 错题链

### 目标
把错误作答沉淀为后续复盘对象。

### 主对象
- `UserError`

### 来源
- `PracticeRecord`
- `PaperPracticeSession` 中完成后的错误题

### 写入
- `UserError`

### 规则
- 错题是派生数据，不是主数据源
- 禁止跳过练习记录，直接把页面行为写成错题
- 错题回跳原题时，应以 `Question` 为准

---

## 5. 复盘链（预留）

### 目标
从错题走向再练与复盘。

### 当前建议
- 仍以 `UserError` 为主入口
- 后续再扩展 review / schedule / note 等结构
- 现阶段不要为了复盘先新增一套主表抢占主链

---
