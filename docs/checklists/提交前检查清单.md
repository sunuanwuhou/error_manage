# 提交前检查清单

## 一、模型检查
- [ ] 这次改动是否仍然使用唯一主模型
- [ ] 有没有引入同义新模型
- [ ] 有没有出现同一字段多处不同语义

## 二、链路检查
- [ ] 这次改动属于哪一条主链
- [ ] 是否跨了两条以上主链
- [ ] 是否补了“主链映射”

## 三、改动顺序检查
- [ ] 是否先改 schema，再改 service / route / page
- [ ] 是否有页面先行、schema 后补的情况
- [ ] 是否把复杂业务逻辑放进了 service 层

## 四、接口检查
- [ ] route 入参是否做了校验
- [ ] 返回结构是否稳定
- [ ] 页面消费的字段是否和 route 返回一致

## 五、测试检查
- [ ] `npm run typecheck`
- [ ] `npm run smoke:backend`
- [ ] `npm run smoke:frontend`

## 六、提交边界检查
- [ ] 本次 commit 是否只主改一段链路
- [ ] 是否夹带了“顺手修”的无关改动
- [ ] 是否能用一句话说清这次提交做了什么

## 七、合并前备注
请在 PR / 提交说明里补充：

```md
本次主链：
主模型：
新增字段：
新增接口：
影响页面：
回归结果：
```
