# AI 分析飞轮

> 理论 → 实践 → 实践反哺理论 的持续进化循环

---

## 飞轮结构

```
第一层：系统给的理论（通用，所有人一样）
  interval = [1,3,7,15,30]
  stockifyProb(60-80%) = 0.8
  speedLimit(判断推理) = 90s

        ↓ 用户实践产生数据 ↓

第二层：ActivityLog 积累原始行为数据
  每次答题、每次遗忘、每次进步

        ↓ 分析服务定期分析 ↓

第三层：AI 从数据提炼个人规律
  "你的判断推理遗忘速率比均值快2.3倍"
  "你在距考试30天时数量关系应该放弃"
  "你的高效时段是上午9-11点"

        ↓ SystemInsight 写入 ↓

第四层：个人理论反哺系统行为
  个性化 interval [1,2,5,12,25]
  个性化 errorROI 权重调整
  个性化每日任务结构
```

---

## 防幻觉机制

AI分析结果不可直接信任，需要置信度校准：

```
AI Finding（初始confidence=0.7）
    ↓
用户在讨论页面：
  ✅ 确认 → confidence +0.15，标记"用户验证"
  ❌ 否定 → confidence=0，记录否定原因
  💬 补充 → 追加上下文

讨论记录写入 AnalysisSnapshot.discussion
    ↓
下次分析：AI读到"上次说X，用户说不对，原因Y"
    → 不重复相同幻觉
    → 置信度随验证次数累积提升
```

**讨论数据本身是训练数据**，越多用户确认/否定，分析质量越高。

## 飞轮启动时间线

```
Week 1:  导入真题 → AnalysisQueue写入 → 等待分析服务
Week 2:  50+错题  → 分析服务首次运行 → AnalysisSnapshot_001
Week 4:  200+事件 → 二次分析（含prev上下文）→ 质量提升
Month 2: 1000+事件→ 个性化interval建议 → 系统行为改变
Month 3: 系统已"认识"这个用户，预测准确率>80%
```
