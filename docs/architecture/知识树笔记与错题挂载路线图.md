# 知识树笔记与错题挂载路线图

Updated: 2026-03-20 14:05 CST

## 背景

当前系统的 `笔记 / 规律` 已经能用，但从真实使用视角仍然有两个核心问题：

1. 内容组织方式仍偏“卡片堆积”，随着使用时间增长会变乱。
2. 错题、AI 分析、笔记之间缺少统一主对象，用户很难形成稳定的知识复盘路径。

因此后续不再把“笔记”和“规律”视为两个平行系统，而是统一收束到“知识点”这一层。

## 目标

把当前的“错题系统 + 笔记区”升级成“知识点驱动的个人提分系统”。

用户看到和使用的主结构应当是：

- 一级模块
- 二级模块
- 三级模块
- 四级 / 具体知识点
- 知识点下挂错题数量与错题列表

## 目标结构

### 1. 知识树结构

推荐的最小稳定结构：

- 一级：题型大类
  - 判断推理
  - 言语理解
  - 数量关系
  - 资料分析
  - 常识判断
- 二级：模块
  - 例如 图形推理 / 定义判断 / 主旨概括 / 工程问题
- 三级：子模块
  - 例如 立体截面 / 因果削弱 / 语句衔接 / 比重增长
- 四级：具体知识点
  - 例如 “立体图形截面判断”
  - 例如 “因果链中的他因削弱”

### 2. 知识点页面应该包含

每个知识点节点显示：

- 知识点名称
- 对应错题数量
- 最近更新时间
- 最近一次挂入的错题来源

点击知识点后展开：

- 知识点正文（Markdown）
- 关联错题列表
- AI 生成的行动规则 / 错因摘要
- 去练同类题入口
- 编辑知识点入口

## 自动沉淀规则

错题做错后，系统自动执行：

1. 根据 `type / subtype / skillTags / AI 错因` 推断目标知识点路径
2. 先查重
   - 若已存在同知识点：不重复创建
   - 只把当前错题挂到该知识点下
3. 若不存在
   - 自动创建“知识点草稿”
   - 预填内容包括：
     - 错因摘要
     - 一句行动规则
     - 题目来源
4. 用户后续再补充为正式知识点

核心原则：

- 知识点是主对象
- 错题是挂载对象
- 重复知识点只更新，不重复新增

## 数据模型演进

### 当前阶段（已开始）

在 `UserNote` 上先补最低可用的知识树字段：

- `type`
- `module2`
- `module3`
- `title`
- `content`
- `sourceErrorIds`

这使得知识页已经可以按“一级 / 二级 / 三级 / 知识点”浏览和筛选。

### 下一阶段（推荐）

新增正式知识点模型，例如：

- `KnowledgeNode`
  - `id`
  - `userId`
  - `level1`
  - `level2`
  - `level3`
  - `level4`
  - `title`
  - `contentMd`
  - `summary`
  - `sourceCount`
  - `masteryStatus`
  - `updatedAt`
- `KnowledgeNodeError`
  - `id`
  - `knowledgeNodeId`
  - `userErrorId`
  - `questionId`
  - `createdAt`

这样后续可以自然支持：

- 知识点下错题数统计
- 点击数字查看错题
- 知识点专项练习
- 知识点掌握度
- AI 知识点总结与变式生成

## 实施顺序

### Phase 1：页面与数据最低可用化

- [x] 笔记页改为“知识笔记”主入口
- [x] `UserNote` 增加 `module2 / module3 / sourceErrorIds`
- [x] 新增知识树筛选与展示
- [x] 练习后可一键沉淀成知识点草稿

### Phase 2：错题挂载

- [x] 为知识点建立“关联错题”最低可用关系（`sourceErrorIds`）
- [x] 知识点详情支持查看错题列表
- [x] 错题做错后自动挂到对应知识点
- [x] 已存在同知识点则跳过创建，只更新关系和计数

### Phase 3：统一知识系统

- [x] 把“规律”收纳为知识点的特殊内容，而不是单独一套导航入口
- [x] 历史规律摘要已并入知识点视图，不再单独展示“规律卡”主区
- [ ] AI 建议统一沉淀到知识点层
- [ ] 知识点支持复习状态、掌握度和专项练习

## 最新进展

- 知识页已改为单轨“知识树”主视图，用户只面对知识点，不再面对“笔记 / 规律”双系统。
- 历史 `UserInsight` 内容仍保留，但现在作为知识点下的“规则摘要 / 易错陷阱 / 公式记忆”辅助呈现。
- 练习页原“固化成规律”入口已改成“补充规则摘要”，落地到知识点草稿，而不是再建独立规律条目。

### Phase 4：提分闭环

- [ ] 基于知识点聚合错因
- [ ] 基于知识点生成行动规则
- [ ] 基于知识点进行变式训练 / 迁移训练
- [ ] 进度页展示“哪些知识点正在补齐”

## 产品判断

这条路线优于当前“笔记 + 规律双轨”的原因：

1. 更符合人脑组织知识的方式
2. 错题终于有明确归宿
3. AI 可以沉淀在知识点层，而不是只解释一题
4. 页面不会随着数据增加而越来越乱
5. 更适合长期提分，而不是短期记录

## 当前结论

后续默认以“知识树 + 错题挂载”作为笔记系统主线。

除非出现明确阻塞，不再回到“笔记 / 规律两个平行系统”的产品结构。
