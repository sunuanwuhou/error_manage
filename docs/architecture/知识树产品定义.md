# 知识树产品定义

> Updated: 2026-03-21  
> 作用：统一知识树、笔记、规则摘要、错题挂载的产品心智。

---

## 1. 核心定义

### 一个知识点 = 一篇 Markdown 笔记

这是当前唯一正确心智。

不是：
- 知识树一套
- 笔记一套
- 规律一套

而是：
- 一级、二级、三级是目录
- 具体知识点本身就是正文对象
- 这篇正文用 Markdown 编辑
- 错题和规则摘要挂在这篇正文下面

---

## 2. 结构

### 层级

1. 一级模块
2. 二级模块
3. 三级模块
4. 具体知识点

### 每个知识点包含

- 标题
- Markdown 正文
- 关联错题
- 规则摘要
- 最近更新时间
- 是否私有

---

## 3. 展示规则

### 默认规则

- 默认只展开一级
- 二级、三级默认收起
- 展开知识点后，直接看到 Markdown 正文
- 不默认展示题目摘录
- 具体题目通过“关联错题 N 道，点击查看”进入

### 编辑规则

- 已有知识点优先页内直接编辑
- 不让用户先跳复杂大表单
- 支持：
  - Markdown
  - 图片粘贴
  - 简单预览

### 错题查看规则

- 单题时直接看该题
- 多题时按一题一题查看
- 不一次性堆满整个弹窗

---

## 4. 数据规则

### 正确关联

- 知识点下的错题数量必须基于真实存在的错题实体
- 失效 `sourceErrorIds` 不能继续当真显示

### 自动沉淀

错题进入系统后：
- 优先归到已有知识点
- 没有则自动生成知识点草稿
- 重复知识点不重复创建

### 规则摘要

- 保留内容价值
- 不再作为独立产品模块
- 只作为知识点下的补充信息存在

---

## 5. 不该继续出现的形态

- `未细分` 这类占位词直接暴露给用户
- “知识点下面再套一层卡片感很重的笔记”
- “先看题目摘录，再猜正文在哪里”
- 把 `规律` 作为与知识点并列的独立入口

---

## 6. 当前仍需继续收口的点

- 失效错题引用清理
- 泛节点治理
- 新增知识点体验轻量化
- 知识点回练入口更自然
- 知识树从“知识管理”继续推进到“学习主场”
