# 数据流设计

> 三层数据结构：原始事件 → AI分析 → 系统反哺

---

## 整体数据流

```
用户操作
  ↓
[Layer 1] ActivityLog        原始事件流（黑匣子，不丢失任何上下文）
  ↓
[Layer 2] AnalysisSnapshot   AI分析结果（带飞轮上下文链）
  ↓
[Layer 3] SystemInsight      反哺系统的可执行建议
  ↓
系统行为改变（interval / errorROI权重 / 推题策略）
```

---

## Layer 1: ActivityLog

**核心原则**：记录过程，不只记结果。

```
现在DB存的：UserError.masteryPercent = 75（只有结果）

ActivityLog存的：
  [答题] 03-01 判断推理 答对 用时45s mastery 60→75
  [答题] 03-03 判断推理 答错 思路wrong mastery 75→55
  [答题] 03-07 判断推理 专项训练 答对 mastery 55→70
  [导入] 03-08 批量导入30道新题
```

**已自动记录的事件**：
- `practice.answer` — 每次答题（含masteryBefore/After/题型/时间/思路判断）
- `error.stockified` — 题目存量化（含从录入到存量化花了几天）
- `import.completed` — 批量导入完成
- `ai.diagnosis_done` — AI诊断完成
- `ai.analysis_done` — 分析服务完成

---

## Layer 2: AnalysisSnapshot

**核心特性**：飞轮上下文链（prevSnapshotId）

```
第1次分析 → AnalysisSnapshot_001
              findings: ["充分必要条件是盲点"]

第2次分析 → AnalysisSnapshot_002
              prevSnapshotId: "001"
              findings: ["充分必要条件改善，削弱题新盲点"]
              （AI看到上次结论，知道什么进步了）

第N次分析 → AnalysisSnapshot_N
              知道完整的学习历史，能预测反弹时机
```

**写入标准接口**（分析服务用）：
```typescript
import { writeAnalysisSnapshot, getPrevSnapshot } from 'src/lib/activity/snapshot-writer'

const prev = await getPrevSnapshot({ userId, analysisType: 'user_pattern' })
const snapshotId = await writeAnalysisSnapshot({
  userId, analysisType: 'user_pattern',
  prevSnapshotId: prev?.id,
  findings: [...],
  recommendations: [...],
  confidenceScore: 0.85,
  dataPointsUsed: 300,
})
```

---

## Layer 3: SystemInsight

**核心特性**：建议可执行，效果可追踪。

```
paramKey="interval_sequence.判断推理"
paramValueOld="[1,3,7,15,30]"
paramValueNew="[1,2,5,12,25]"
status=pending → applied → effectMeasuredAt

effectBefore=0.72  （应用前该题型平均mastery增长率）
effectAfter=0.81   （应用后）
```

高置信度（>0.7）+ 高优先级的建议自动写入此表，待管理员或分析服务应用。

---

## AnalysisQueue: 分析任务调度

```
批量导入完成
    ↓ 按skillTag聚类
AnalysisQueue (pending, priority=frequency)
    ↓ 分析服务按priority消费
AnalysisSnapshot (done)
    ↓
KnowledgeEntry (解法模式)
    ↓ RAG检索
下次AI诊断质量提升
```
