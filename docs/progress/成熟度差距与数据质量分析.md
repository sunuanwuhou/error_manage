# 成熟度差距与数据质量分析

> 更新时间：2026-03-19 18:10 CST
> 目的：从“成熟系统”视角看当前完成度、剩余缺口和数据质量短板

---

## 总体判断

当前项目已经不是“能跑的 demo”，而是**高完成度可用版**。  
如果目标是“成熟、稳定、真正能帮助个人提分的系统”，当前大约在 **94% 左右**，还没有到 100%。

原因不是主流程完全不可用，而是还有几块影响成熟度的短板没有彻底收口：

- 笔记仍偏“便签区”，还不是“学习资产中心”
- 导出仍是第一版备份，不是完整迁移能力
- 系统级自动回归还不够深
- AI 已进入最小闭环，但还没成为真正的策略引擎

## 距离成熟系统还缺什么

### 1. 笔记模块还差最后一跃

当前状态：

- 已支持笔记与规律分栏
- 已支持新增、编辑、删除、搜索、筛选
- 已支持局部失败隔离

还不够成熟的地方：

- 还没有和 `questionId / userErrorId / paperKey` 建立稳定关联
- 还缺“来源”概念：手写、AI 草稿、由错题生成、由套卷复盘生成
- 还缺复习属性：优先级、下次复习时间、已复盘次数、置顶
- 还缺标签体系：题型标签、错因标签、考点标签

结论：

> 笔记已经“能用”，但还没有成为真正提升分数的复盘资产。

### 2. 导出能力仍是第一版

当前状态：

- 能导出用户自己的配置、错题、复习记录、笔记、洞察、模考、分段统计 JSON

缺口：

- 还不是完整迁移包
- 还缺导入失败明细与更可恢复的结构化备份
- 还缺更清晰的版本号 / schema 信息

### 3. AI 已闭环，但还不够“提分引擎”

当前状态：

- `SystemInsight -> daily-tasks` 最小闭环已经打通
- 首页和进度页能看到“今天怎么练、为什么这么练”

缺口：

- 还缺跨题归纳
- 还缺周级策略调整
- 还缺“今天练完涨了多少”的更强反馈闭环
- 还缺知识库预热后的高质量对比验证

### 4. 自动回归还不够深

当前状态：

- 已有 Playwright 冒烟
- 已有 import/OCR 回归

缺口：

- 还缺导入上传 -> 预览 -> 确认 的完整断言
- 还缺设置保存回归
- 还缺笔记新增 / 编辑 / 删除回归
- 还缺每日练习与套卷交卷总结的更完整 E2E

## 数据质量分析

## 当前数据质量优点

- 题库主字段已经够支撑练习、导入、套卷和复盘
- 套卷已有 `paperKey / paperYear / paperProvince / paperExamType`
- 错题、复习记录、模考、分段统计都已有基础结构
- 导入链路已经开始控制重复、低质量题和 OCR 脏数据

## 当前数据质量短板

### 1. 笔记数据结构偏弱

`UserNote` 当前主要字段：

- `type`
- `subtype`
- `title`
- `content`
- `isPrivate`

这对“记录一条文本笔记”够用，但对“长期学习资产”不够。

### 2. 部分核心数据仍是字符串 JSON

例如：

- `Question.options`
- `Question.skillTags`
- `UserError.masteryHistory`
- `PaperPracticeSession.answeredIndices`
- `PaperPracticeSession.markedIndices`
- `PaperPracticeSession.answersJson`

这在 MVP 阶段合理，但对成熟系统的分析能力、质量约束和迁移能力不够友好。

### 3. 导出结构还偏“当前状态”，不够“完整轨迹”

成熟系统更适合导出：

- 数据版本
- 用户配置
- 题库关联
- 错题轨迹
- 练习轨迹
- AI 洞察轨迹
- 笔记来源与应用情况

而不是只导出若干表的当前快照。

- 让 AI 洞察、错题、笔记之间可追踪
- 做更稳定的数据质量统计报表

---

## 建议的下一阶段顺序

1. 继续升级笔记模块，让它从“笔记区”变成“复盘资产中心”
2. 补导入确认、设置保存、笔记 CRUD、每日练习、套卷总结的完整 E2E
3. 增强导出为“可迁移备份”
4. 继续把 AI 从“有建议”推进到“持续提分引擎”

---

## 结论

当前系统已经足够说明方向是对的，也已经具备较强可用性。  
但如果用“成熟系统”的标准衡量，**最后的差距不在页面数量，而在资产化、可靠性和闭环深度**。

其中最值得继续投的点，就是：

- 笔记
- 导出
- 自动回归
- AI 策略闭环
