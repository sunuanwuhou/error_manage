# 待修复池修后放行说明

## 目标
待修复池不能只负责“拦住”，还必须负责把用户重新带回主链。

也就是：
- 看到阻断题
- 修题
- 再次校验
- 条件满足后重新发布到练习流

## 当前阶段实现
本阶段先做轻量闭环，不引入额外复杂状态表：

1. 待修复池可查看某导入任务中的阻断题
2. 可直接跳回导入页定位到某题修复
3. 当某任务已经没有阻断题时：
   - 待修复池显示“可放行”
   - 导入页显示“当前任务已可发布”
   - 用户可直接执行确认入库

## 判断标准
如果用户能够从：
- 待修复池
- 返回导入页
- 看到已无阻断题
- 继续确认入库
那么这条修后放行的主线就是通的。

## 后续可继续升级
- 增加“仅发布当前已修复题”
- 增加阻断题修复历史
- 增加正式修复状态字段
- 增加修复责任人与时间线
