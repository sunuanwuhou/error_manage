# 完成度清单（成熟度矩阵）

> Updated: 2026-03-21  
> 这份文档不再回答“功能有没有做过”，而回答“现在能不能稳定长期用”。

---

## 分级说明

- `A 稳定`：主线可用、心智清楚、数据可信度较高
- `B 可用但粗糙`：能用，但连续使用体验或数据可信度还不够
- `C 未完成`：方向已知，但还没形成可靠产品能力
- `D 已降级/归并`：不再作为独立模块推进

---

## 1. 主流程成熟度

| 主流程 | 当前等级 | 结论 |
|---|---|---|
| 登录 / 鉴权 / onboarding | A | 基本稳定，可继续用 |
| 设置保存 | B | 基本可用，但信息架构仍偏重 |
| 导入题目 / 真题 | B | 多格式已接上，但导入可信度与结果反馈不足 |
| 套卷列表 / 套卷练习 | B | 已进入可用态，但仍不够像成熟考试产品 |
| 普通练习 | B | 已有多模式与跳题，但节奏与后续动作还不够顺 |
| 错题沉淀 | B | 能沉淀，但与知识树的数据一致性仍需收口 |
| 知识树 / 笔记 | B | 方向已明确，但数据质量和编辑体验仍需继续打磨 |
| 进度 / 策略反馈 | B | 已有进度与策略卡，但行动导向还不够强 |
| 导出 | C | 当前更像备份，不是完整迁移能力 |

---

## 2. 当前已稳定的部分

### A 稳定

- 本地 PostgreSQL 开发主线
- `master` 分支作为当前主线
- Next.js / Prisma / NextAuth 基础运行框架
- 导入基础设施：
  - Excel
  - DOCX
  - PDF
  - 截图 OCR 入口
- 练习基础设施：
  - 快速练习
  - 深度练习
  - 套卷练习
  - 聚焦 / 计时训练
- 基础工程检查：
  - `npm run typecheck`
  - `npm run build`

---

## 3. 可用但粗糙的部分

### B 可用但粗糙

#### 导入

- 文件解析已经可用
- 但还差：
  - 导入结果页更可信
  - 重复题策略可见化
  - 同一套卷导入后的追踪页
  - 多格式结果一致性验证

#### 套卷

- 已有题卡、继续作答、总结
- 但还差：
  - 列表页完成态 / 恢复态
  - 更强的考试节奏感
  - 与知识点回练的衔接

#### 普通练习

- 模式和基本交互已在
- 但还差：
  - 更顺的“做完题 -> 沉淀 -> 继续练”链路
  - 和套卷、知识点回练的统一心智

#### 知识树 / 笔记

- 路线已定为：
  - 一个知识点 = 一篇 Markdown 笔记
  - 错题挂在知识点下面
- 但还差：
  - 失效 `sourceErrorIds` 清理
  - 真实错题关联修复
  - 泛节点治理
  - 新增/编辑知识点体验统一

#### 进度页 / 首页

- 已经有策略和预测基础
- 但还差：
  - 更强的主行动
  - 更清晰的“今天练了涨多少”
  - 更直接的差距来源表达

---

## 4. 未完成但明确要做的部分

### C 未完成

- 导入完成后的“结果追踪页”
- 知识树数据修复脚本和治理规则
- 真正统一的训练闭环：
  - 普通练习
  - 套卷练习
  - 知识点回练
- 全链路 smoke 执行纪律
- 完整迁移级导出

---

## 5. 已降级或归并的方向

### D 已降级 / 归并

- `规律` 不再作为独立产品系统推进
  - 现在只保留内容价值
  - 已归并进知识点 / 规则摘要体系

- AI 相关高级能力本轮不继续作为主开发项推进
  - 保留路线图
  - 只做移交文档和边界说明

---

## 6. AI 当前口径

当前 AI 能力分两层：

### 保留

- AI 失败时不炸主流程
- 单题诊断入口
- 规则摘要落点清晰
- 文档化责任边界

### 移交，不在当前轮深做

- `analysis-worker` 深化
- RAG 优化
- 向量召回增强
- 规则效果评分闭环
- 更复杂 OCR / 图题理解增强
- 多题归纳与社区洞察

---

## 7. 当前判断

如果以“成熟可长期用的提分系统”为标准：

- 整体成熟度：`70%-80%`
- 当前最值得继续投入的 3 条主线：
  1. 导入可信度
  2. 知识树数据与体验收口
  3. 训练闭环统一

换句话说：

**系统已经不是 MVP，但还没有成熟到可以长期稳定托管用户备考过程。**
