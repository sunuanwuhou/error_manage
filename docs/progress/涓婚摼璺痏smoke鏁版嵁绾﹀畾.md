# 主链路 Smoke 数据约定

## 目标
让浏览器级 smoke 不再只停在“页面能打开”，而是逐步接入：
- 真实存在的测试账号
- 真实存在的题目样本
- 真实存在的套卷/错题数据

## 约定 1：测试账号
建议固定一个 smoke 账号：
- 仅用于本地/测试环境
- 具备管理员或最小验证权限
- 可以访问：
  - /import
  - /papers
  - /practice
  - /wrong-questions

## 约定 2：最小题目样本
建议准备一套最小样本：
- 至少 3 道题
- 至少 1 道有解析
- 至少 1 道带图片
- 至少 1 道会进入错题本

## 约定 3：最小套卷样本
建议固定 1 套 smoke 套卷：
- 命名稳定
- questionId 稳定
- 允许重复重建
- 允许删除重置

## 约定 4：导入样本
建议固定 1 个 DOCX 样本：
- 文件名带 年份 / 省份 / 考试类型 / 场次
- 可以触发自动识别
- 可以触发原文对照
- 可以触发自动校对建议

## 验收标准
smoke 至少要能稳定验证：
1. 导入页可打开
2. 套卷可打开
3. 练习可提交
4. 结果页可见
5. 错题页可达
6. 回看入口可达
