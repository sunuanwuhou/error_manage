## 本轮变更主题

### 1. 套卷练习从“普通练习换皮”推进到“考试流”

目标：

- 让套卷成为独立语义的练习模式，而不是普通刷题页的轻改版。

主要变化：

- 引入套卷列表、套卷详情、套卷会话持久化相关页面和 API。
- 套卷支持开始、继续作答、交卷检查、总结页回看。
- 增强题号感、跳题、存疑标记、交卷前检查。
- 套卷答题与普通练习进一步分流。
- 总结页补充整卷复盘、模块表现、优先回看模块、下一步建议。

补充修正：

- 套卷模块识别不再直接信任单题瞬时类型。
- 当前已引入模块平滑纠偏规则，缓解少量题目误判导致的模块碎片化。
- 交卷检查与总结页已统一使用同一套模块口径。

主要影响范围：

- `src/app/(app)/practice/page.tsx`
- `src/app/(app)/practice/summary/page.tsx`
- `src/app/(app)/papers/`
- `src/app/api/papers/`
- `src/app/api/paper-sessions/`
- `src/app/api/review/session-summary/route.ts`
- `src/lib/papers.ts`
- `src/lib/paper-modules.ts`

### 2. 导入主链路收口，减少“看起来成功、实际没进系统”

目标：

- 让文件导入、截图导入、OCR 识别最终都走同一条正式确认入库链路。

主要变化：

- 导入确认逻辑增强，补齐预览勾选尊重、真实落库与去重策略。
- 图片 / OCR 导入从“识别后单独分叉处理”收回到正式导入确认流程。
- 重复题不再只有隐式跳过，新增更明确的重复处理策略。
- 导入时的索引错位、预览编辑不入库等问题被修复。
- 补充导入/OCR 相关工具与测试。

主要影响范围：

- `src/app/(app)/import/page.tsx`
- `src/app/(app)/import/screenshot/page.tsx`
- `src/app/api/import/upload/route.ts`
- `src/app/api/import/confirm/route.ts`
- `src/app/api/import/screenshot/route.ts`
- `src/app/api/ai/ocr/route.ts`
- `src/lib/import/`
- `src/lib/parsers/excel-parser.ts`
- `src/lib/parsers/pdf-parser.ts`
- `src/lib/parsers/docx-parser.ts`

### 3. 配置完成态和 onboarding 收口，减少反复打扰

目标：

- 已经完成配置的用户，不应该在主流程中反复被拉回配置。

主要变化：

- 配置完成态不再依赖松散启发式，而是收敛到更明确的完成标识。
- onboarding 提交开始真实处理失败分支，不再默认假成功跳转。
- 主 layout 和 onboarding API 对历史账号做兼容补齐。
- 设置页继续作为完整配置入口，其余流程尽量静默带入已保存配置。

主要影响范围：

- `src/app/(app)/layout.tsx`
- `src/app/(app)/onboarding/page.tsx`
- `src/app/(app)/settings/page.tsx`
- `src/app/api/onboarding/route.ts`
- `src/lib/auth.ts`
- `prisma/schema.prisma`

### 4. 进度页、首页与 AI 训练策略形成最小闭环

主要变化：

- 首页开始展示当前训练策略摘要。
- `SystemInsight` 与日常训练策略之间开始建立最小回路。
- 分析讨论页支持对系统建议进行处理。

主要影响范围：
