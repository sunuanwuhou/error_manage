# 导入成卷直练闭环说明

## 问题
此前主链中存在一个明显断点：

- 题目导入成功
- 题目进入题库
- 但用户不知道接下来该怎么练这一套卷

也就是说：
“题进库了” 不等于 “卷成了并可直接练”。

## 当前阶段目标
先不引入复杂的额外套卷结构，而是基于当前已经稳定的 `paperKey` 规则，
把“导入成功 → 形成套卷口径 → 直接进入练习”打通。

## 当前闭环
导入确认成功后，结果区必须明确给出：
- 这套卷对应的 paperKey
- 去练这套卷
- 去练这套卷（限题量快速验证）
- 返回练习页

## 为什么先这样做
当前最重要的是让主线顺下来，而不是先做一层很重的卷管理系统。

在已有前提下：
- `srcYear`
- `srcProvince`
- `examType`
- `srcSession`
已经足以形成一个稳定的套卷口径入口。

## 后续可以继续升级的方向
- 导入任务与正式试卷实体绑定
- 明确“本次导入形成了哪一套卷”
- 卷级统计、题量、发布状态
- 卷级修题与卷级练习调度
