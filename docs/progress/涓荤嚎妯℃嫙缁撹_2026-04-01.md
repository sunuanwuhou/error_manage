# 主线模拟结论（2026-04-01）

## 模拟范围
本次不是只看单点功能，而是按当前项目文档定义的主线，完整模拟：

真题导入 → 题库入库 → 刷题作答 → 自动判分 → `error_analysis` 式错题管理 → 笔记/知识树联动 → 复盘训练

并结合当前 patch 状态，按真实用户路径检查：

1. 上传真题
2. 解析预览
3. 校对
4. 入库
5. 开始练习
6. 提交答案
7. 进入错题工作台
8. 写笔记 / 挂知识点
9. 再练 / 复盘

---

## 一、当前已经基本打通的部分

### 1. 导入前半段已经基本成型
当前已经具备：
- 文件上传
- 文件名自动识别年份 / 省份 / 考试类型
- 预览校对
- 自动校对建议
- 原文对照
- 重复导入预检查
- 导入质量门禁
- 服务端再次拦截坏题

阶段判断：
导入链已经不再只是“上传文件”，而是进入“预览 / 校对 / 拦截坏题 / 再入库”的状态。

### 2. 练习页到结果页已经能跑通
当前已经具备：
- 开始练习
- 作答
- 判分
- 正确率
- 错题摘要
- 回看本轮
- 去错题工作台 / 错题本
- 再练一道错题
- 独立结果页

阶段判断：
“刷题作答 → 自动判分 → 结果反馈”这段已经具备主链雏形。

### 3. 错题工作台已不再只是展示页
当前已经具备：
- 错题工作台入口
- 题型分布
- 错因分布
- 待复习 / 低掌握度过滤
- 手动录入错题
- 错题—笔记联动页
- 笔记编辑入口
- 知识点挂接入口
- 草稿箱
- 本地草稿状态
- 服务端保存 / 回显
- 已保存 / 已完成状态

阶段判断：
“错题 → 笔记 → 知识点”已经开始从页面入口变成可持续处理流。

---

## 二、当前真正还卡主线的 6 个断点

### 断点 1：导入后的“成卷 / 归卷”还不够明确
问题：
- 题进库了，不等于卷成了
- 用户不知道导入成功后该去哪练这套卷
- 套卷形成与导入成功之间缺少更强的显式闭环

结论：
必须补“导入成功 → 形成套卷 → 去练这套卷”的闭环入口。

### 断点 2：质量门禁有了，但“待修复池”还没完整落地
问题：
- 坏题能拦
- 但被拦后的题去哪修、怎么修、修完怎么再放行，还不够正式

结论：
需要把阻断题从导入页临时状态，升级成正式待修复池。

### 断点 3：错题工作台状态还没反向影响训练调度
问题：
当前状态更多是“能展示”，还没有真正影响：
- 再练顺序
- 复盘优先级
- 下一轮训练建议

结论：
错题工作台中的状态，必须开始反向影响下一步训练。

### 断点 4：知识点挂接还只是记录，不是真正知识树联动
问题：
当前能挂模块名、节点名、原因，但还没有：
- 节点视图
- 节点下错题聚合
- 节点下笔记
- 节点下再练入口

结论：
下一阶段要把“挂知识点”升级成真正知识树联动。

### 断点 5：当前服务端记录层还是兼容层
问题：
当前后半段服务端记录采用兼容存储，不适合作为长期正式结构。

结论：
后续必须迁到正式表结构。

### 断点 6：还缺一条默认主路径
问题：
现在功能很多，但用户做完题后路径仍偏散：
- 看结果
- 去回看
- 去错题工作台
- 去错题本
- 再练

缺少系统明确带路的一条默认主路径。

结论：
结果页应逐步收成唯一主路径：
做完题 → 看结果 → 进入错题工作台 → 补笔记 / 挂知识点 → 再练

---

## 三、主线本身不用再改

当前主线已经清楚，不需要再改成别的：

真题导入 → 题库入库 → 刷题作答 → 自动判分 → `error_analysis` 错题工作台 → 笔记/知识树联动 → 复盘训练

问题不在主线方向，而在每一段要从“能用”继续收成“默认顺着走”。

---

## 四、当前阶段的一句话结论

当前最缺的不是更多页面，而是这三件事：
1. 导入后直接成卷可练
2. 坏题进入正式待修复池
3. 错题工作台状态真正驱动下一步训练
