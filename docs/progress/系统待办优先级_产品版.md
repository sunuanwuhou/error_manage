# 系统待办优先级（产品版）

> Updated: 2026-03-21  
> 只回答一件事：现在还缺什么，为什么先做它。

---

## P0：不补就不该叫成熟系统

### P0.1 导入结果可信度

为什么先做：
- 这是用户信任系统的入口
- 如果导入结果不透明，后面套卷、知识树、AI 都站不住

要做：
- 导入结果页
- 重复题策略可见化
- 同套卷导入结果可追踪
- PDF / DOCX / Excel 一致性验证

### P0.2 知识树数据修复

为什么先做：
- 当前知识树方向对，但旧数据有断链
- 如果不先修数据，页面越做越会误导用户

要做：
- 失效 `sourceErrorIds` 清理
- 能回填的错题关联回填
- 泛节点治理
- 历史规则与知识点的归并口径统一

### P0.3 练习闭环统一

为什么先做：
- 现在普通练习、套卷、知识点回练还像三条线
- 成熟系统必须让训练心智统一

要做：
- 练习结束后的下一步统一
- 知识点回练更自然
- 套卷总结到知识点/错题回流更清楚

### P0.4 全链 smoke 真正落地

为什么先做：
- 不跑全链路，主流程回归会反复发生

要做：
- 固化 smoke 入口
- 关键链路最小 E2E
- 把“文档要求”变成“执行纪律”

---

## P1：影响长期使用体验和提分效率

### P1.1 Dashboard 行动化

要做：
- 首屏只强调“今天该做什么”
- 压缩解释型信息
- 更快进入练习

### P1.2 Stats 提分驾驶舱化

要做：
- 保守/乐观预测更强呈现
- 今天练了涨多少
- 差距来源具体到模块/知识点
- 下一步建议更可执行

### P1.3 Papers 考试产品化

要做：
- 列表页显示进度和恢复状态
- 更强的完成态提示
- 总结页模块回练入口

### P1.4 Settings 信息架构收缩

要做：
- 高频设置和低频功能分层
- 降低后台感
- 不再让设置承担太多入口职责

### P1.5 知识树学习主场化

要做：
- 今日最该回看的知识点
- 最近连续出错知识点
- 待练感、掌握状态更可信

## AI：只写文档移交，不在当前轮深做

### 只保留边界

- AI 失败不炸主流程
- AI 产出落点明确
- AI 责任边界文档化

### 不继续在当前轮深做

- `analysis-worker` 深化
- RAG / 向量召回优化
- 规则效果评分闭环
- 更复杂 OCR / 图题理解
- 多题归纳、社区洞察、蒸馏
