# 主链路 Smoke 计划

## 目标
把“真题导入 -> 题库入库 -> 刷题作答 -> 自动判分 -> 错题沉淀 -> 复盘训练”
收成一组固定的最小浏览器级 smoke。

## 先做的 3 条
1. 导入 -> 入库 -> 导入结果可见
2. 练习 -> 提交 -> 判分 -> 结果页
3. 错题 -> 原题 -> 再练 -> 回看

## 验收原则
- 失败时必须可见，不允许页面静默
- 成功后必须有明确跳转或结果反馈
- 关键统计必须可见：
  - 导入结果
  - 正确率
  - 错题数量
  - 回看入口

## 为什么现在做
主流程已经基本可用，但如果没有固定 smoke，后续每次 patch 都可能把主链打断。
